What Could Possibly Go Wrong?

headers already sent

Most web frameworks will blow up if you try to set a HTTP Header after you have started to send a response body.

Common Error Codes

  • Can't render headers after they are sent to the client.
  • Cannot modify header information - headers already sent

Troubleshooting

General

Search through your source code for APIs that directly or indirectly set HTTP headers. Setting the Content-Type on the HTTP response is an obvious example. An indirect one is using libraries that change the session. Session changes probably mean setting the Cookie, which is done via a HTTP header.

Another common source of the error is printing to standard out to debug, early in request processing. Search for raw output, instead of proper logging APIs.

NodeJS

APIs that write headers

res.setHeader(name, value);
res.writeHead(statusCode);

APIs that write the body

res.write(data);
res.end(data);

PHP

APIs that write headers

header();
setcookie();
session_start();
